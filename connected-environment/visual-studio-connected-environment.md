# <a name="vs-connected-environment-for-azure-container-service-aks-preview"></a>VS Connected Environment para Azure Container Service (AKS) (Preview)
VS Connected Environment ayuda a desarrollar a gran velocidad en Kubernetes. Primero se crea un entorno de Kubernetes totalmente administrado a través de Azure y optimizado para el desarrollo y, seguidamente, se desarrollan contenedores de forma iterativa en la nube recurriendo a herramientas conocidas como VS Code, Visual Studio o la línea de comandos.

Este método aporta varias ventajas:

* Se obtiene un entorno de desarrollo sin infraestructura y representativo de la producción, con acceso completo a los recursos de nube.
* Los contenedores de Node.js y .NET Core se depuran directamente en Kubernetes con VS Code o Visual Studio. Todos los demás lenguajes se pueden desarrollar con la interfaz de línea de comandos.
* Una instancia de Kubernetes se puede compartir con todo el equipo de desarrollo para ahorrar costos y minimizar la tarea de configuración del equipo local para los nuevos miembros del equipo.
* Se puede desarrollar código de forma aislada y realizar pruebas de un extremo a otro con otros componentes sin tener que replicar ni simular dependencias.
