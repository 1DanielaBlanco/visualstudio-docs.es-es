# C26167
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

Advertencia C26167: Posiblemente se liberó un bloqueo no retenido \<bloqueo\> en la función \<función\>.  
  
 La advertencia C26167 se parece a la advertencia [C26117](../code-quality/c26117.md) salvo que el nivel de confianza es inferior.  Por ejemplo, la función puede contener errores de anotación.  
  
## Ejemplo  
 El código siguiente generará C26167, así como C26110.  
  
```cpp  
  
typedef struct _DATA {   
    CRITICAL_SECTION cs;   
} DATA;   
  
_Releases_lock_(p->cs) void Leave(DATA* p) {   
    LeaveCriticalSection(&p->cs); // OK   
}   
void ReleaseUnheldLock(DATA* p) { // Warning C26167  
    int i = 0;  
    Leave(p); // Warning C26110  
}  
  
```  
  
## Ejemplo  
 El código siguiente corregirá estas advertencias.  
  
```cpp  
  
typedef struct _DATA {   
    CRITICAL_SECTION cs;   
} DATA;   
  
_Releases_lock_(p->cs) void Leave(DATA* p) {   
    LeaveCriticalSection( &p->cs );  
}   
  
void ReleaseUnheldLock( DATA* p ) {  
    EnterCriticalSection( &p->cs );  
    int i = 0;  
    Leave(p);  
}  
  
```
