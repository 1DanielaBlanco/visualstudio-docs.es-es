# <a name="javascript-representation-of-windows-runtime-types"></a>Representación de JavaScript de tipos de Windows Runtime
En la tabla siguiente se describe el modo en que JavaScript representa los tipos de Windows Runtime.  
  
> [!IMPORTANT]
>  Las características de Windows en tiempo de ejecución no están disponibles para las aplicaciones que se ejecutan en Internet Explorer.  
  
## <a name="windows-runtime-types-in-javascript"></a>Tipos de Windows en tiempo de ejecución en JavaScript  
  
|Tipo de Windows en tiempo de ejecución|Representación en JavaScript|Descripción|  
|--------------------------|-------------------------------|-----------------|  
|`UInt8`|`Number`|Un `Uint8` de Windows en tiempo de ejecución es un entero de 8 bits sin signo, representado como un número de JavaScript. Un valor de JavaScript primero se convierte en un número de JavaScript y, a continuación, se sigue el proceso `ToUint32`. Si el valor del número de JavaScript está fuera del intervalo de un Uint8, el resultado tendrá aplicado el módulo 2^8.|  
|`Int32`|`Number`|Un `Int32` de Windows en tiempo de ejecución es un entero de 32 bits con signo, representado como un número de JavaScript. Un valor de JavaScript primero se convierte en un número de JavaScript y, a continuación, se sigue el proceso `ToInt32`. Si el valor del número de JavaScript está fuera del intervalo de un `Int32`, el resultado tendrá aplicado el módulo 2^16.|  
|`Int64`|`Number`|Un `Int64` de Windows en tiempo de ejecución es un entero de 64 bits con signo, representado como número estándar si entra dentro del intervalo [- 2^53, 2^53]. Si sale fuera de este intervalo, se representa como valor del número con una memoria auxiliar personalizada que mantiene los 64 bits completos de datos enteros. Todas las operaciones matemáticas en estos valores de número Int64 personalizados hacen que el valor se convierta en un número estándar en el intervalo [- 2^53, 2^53]. Si el valor está fuera de este intervalo, se produce un error de tipo. Las excepciones a este proceso de conversión son `==`, `===`, `SameValue` y las operaciones de `RelationalComparison`, que comparan los argumentos basados en los 64 bits completos cuando se pasan dos valores de 64 bits representados. Si el argumento es un número estándar de JavaScript, estas operaciones también convierten el argumento en un número de JavaScript antes de la comparación. Un valor de JavaScript se asigna directamente si es el propio valor Int64; de lo contrario, es el resultado de aplicar la conversión `ToInteger` en el valor que se pasa a Windows en tiempo de ejecución. Si se produce un error en la conversión de tipos, se devuelve una excepción.|  
|`Uint64`|`Number`|Un `Uint64` de Windows en tiempo de ejecución es un entero de 64 bits sin signo, representado como número estándar si entra dentro del intervalo [0, 2^53]. Si sale fuera de este intervalo, se representa como un número con una memoria auxiliar personalizada que mantiene los 64 bits completos de datos enteros sin signo. Todas las operaciones matemáticas sobre estos valores Uint64 personalizados hacen que el valor se convierta en un número estándar en el intervalo [0, 2^53]. Si el valor está fuera de este intervalo, se produce un error de tipo. Las excepciones a este proceso de conversión son `==`, `===`, `SameValue` y las operaciones de `RelationalComparison`, que comparan los argumentos basados en los 64 bits completos cuando se pasan dos valores de 64 bits. Si el argumento es un número estándar de JavaScript, estas operaciones también convierten el argumento en un número de JavaScript antes de la comparación. Un valor de JavaScript se asigna directamente si es el propio valor Uint64; en caso contrario, el resultado de aplicar la conversión `ToInteger` en el valor, seguido del módulo 2^52, se pasa a Windows en tiempo de ejecución. Si se produce un error en la conversión de tipos, se devuelve una excepción.|  
|`Single`|`Number`|Un `Single` de Windows en tiempo de ejecución es un número de punto flotante de 32 bits, representado como un número de JavaScript escogiendo el valor de precisión doble más cercano al valor de precisión simple. Un valor de JavaScript se convierte en un número de JavaScript y, a continuación, el intervalo se comprueba para garantizar que el valor se puede representar en un número de punto flotante de precisión simple IEEE 754 de 32 bits. Si se produce un error en la conversión o la comprobación del intervalo, se devuelve un error de serialización.|  
|`Double`|`Number`|Un `Double` de Windows en tiempo de ejecución es un número de punto flotante de 64 bits. `ToNumber` se usa para convertir un `Double` de Windows en tiempo de ejecución en un número de JavaScript. Si se produce un error en la conversión, se devuelve un error de serialización.|  
|`Boolean`|`Boolean`|Un `Boolean` de Windows en tiempo de ejecución es un valor de 8 bits donde cero es `false` y cualquier valor distinto de cero es `true`, representado como `Boolean` de JavaScript. Un valor de JavaScript primero se convierte en `Boolean` de JavaScript mediante `ToBoolean`. Esto significa que una función de Windows en tiempo de ejecución declarada como `void func(BOOL);` se podría escribir en JavaScript como `obj.func("test");`. Se llama a la función de Windows en tiempo de ejecución con el parámetro `BOOL` pasado como `TRUE`. Si se produce un error en la conversión, se devuelve un error de serialización.|  
|`Char16`|`String`|Un `Char16` de Windows en tiempo de ejecución es un carácter Unicode de 16 bits, representado como una cadena de JavaScript de un único carácter. Un valor de JavaScript se convierte en una cadena de JavaScript mediante `ToString` y después se comprueba para garantizar que la cadena es de un solo carácter. El carácter individual, a continuación, se pasa como el valor `Char16`. Si se produce un error en la conversión o la comprobación de la longitud, se devuelve un error de serialización.|  
|`String`|`String`|Un `String` de Windows en tiempo de ejecución es una secuencia inmutable de objetos `Char16`. Las cadenas se representan como objetos `String` de JavaScript. Las cadenas de Windows en tiempo de ejecución no tienen valores diferentes para `null` y la cadena vacía (""). `null` y los valores de cadena vacía se convierten en la cadena vacía de JavaScript. Nota: cuando `null` de JavaScript se pasa a un parámetro de Windows en tiempo de ejecución que espera una cadena, genera el valor de cadena “null”, no `null` o la cadena vacía (""). Las cadenas que pasan a Windows Runtime siempre deberían crear instancias en la cadena vacía.|  
|`Enumeration`|`Object`|Un `Enumeration` de Windows en tiempo de ejecución es un entero de 32 bits (con o sin signo) que tiene un conjunto asociado de constantes con nombre. En JavaScript, una enumeración se representa como un objeto que contiene un campo de solo lectura para cada valor con nombre. Los valores de enumeración se convierten en números de JavaScript como se ha definido anteriormente para `Int32` y `UInt32`. Cuando un valor de JavaScript se convierte en una enumeración de Windows en tiempo de ejecución, se convierte, se trunca y se comprueba el intervalo como se describió anteriormente. Sin embargo, el valor resultante no se comprueba en los valores con nombre definido que se especifican en la enumeración.|  
|`Structure`|`Object`|Un `Structure` de Windows en tiempo de ejecución es una colección heterogénea de campos de datos con nombre. En JavaScript, una estructura se representa como un objeto de JavaScript que contiene una propiedad con nombre para cada campo de la estructura. Si se produce un error en la conversión de cualquier valor de la estructura, se devuelve un error de cálculo de referencias. Los campos del objeto de JavaScript que no tienen equivalentes en la estructura de Windows en tiempo de ejecución se omiten. Nota: no se pueden crear instancias en una estructura de Windows en tiempo de ejecución en JavaScript con la palabra clave `new`.|  
|`Array`|`Array`|Un `Array` de Windows en tiempo de ejecución se convierte en una matriz de JavaScript. Sin embargo, el tamaño de las matrices de Windows en tiempo de ejecución no se puede cambiar, por lo que algunas operaciones de matriz de JavaScript no son posibles. La propiedad interna [[Class]] de una matriz convertida no se establece en “Matriz” porque esto no se permite en la especificación de ECMAScript 5. Esto significa que `Array.isArray(v)` devuelve `false`. Un valor de JavaScript se convierte en una matriz de Windows en tiempo de ejecución de la siguiente manera: si el valor es `null` o `undefined`, se convierte en `null` nativo. Si su propiedad interna [[Class]] es "Matriz", se copia en una matriz nativa y se pasa una referencia a esta matriz. Si es una matriz convertida, como se ha descrito previamente, se pasa la matriz nativa subyacente. Nota: pasar un valor de matriz de JavaScript a un método de Windows en tiempo de ejecución genera una copia completa de la matriz.|  
|Delegado (devolución de llamada de función)|`Function`|Un delegado de Windows Runtime es una referencia a un único método. Un delegado de Windows en tiempo de ejecución se representa en JavaScript como un objeto `Function` de JavaScript, lo que garantiza que se llame en el subproceso adecuado. Cuando se invoca a `Function`, los argumentos se convierten en los tipos de parámetros equivalentes de Windows en tiempo de ejecución. Si hay menos argumentos de JavaScript que parámetros `in` de delegado, la llamada a delegado da error. Si hay más argumentos de JavaScript que parámetros `in` en el delegado, se omiten los argumentos adicionales de JavaScript. Después de invocar al delegado, los parámetros `out` se convierten en tipos de JavaScript y se devuelven. Si un objeto de función nativo de JavaScript se convierte en un delegado de Windows en tiempo de ejecución, se ajusta en un delegado de Windows en tiempo de ejecución del tipo correspondiente. Cuando se invoca el delegado, los parámetros `in` se convierten en tipos de JavaScript. Después de invocar al delegado, el valor devuelto se convierte en los parámetros `out` del delegado.|  
|Interfaz|`Object`|Interfaces y grupos de interfaces no están directamente representados como objetos en JavaScript. Sin embargo, las interfaces se representan como parámetros y tipos devueltos de los métodos de Windows en tiempo de ejecución. Una instancia de interfaz de Windows Runtime se convierte como sigue:<br /><br /> 1.  Si hay una clase en tiempo de ejecución válida, representa el objeto como instancia de esa clase.<br />2.  Si no hay ninguna clase válida en tiempo de ejecución, representa el objeto como instancia de una clase en tiempo de ejecución sin nombre que implementa exactamente la interfaz que se sabe que la instancia va a implementar, así como sus interfaces necesarias.<br /><br /> Un valor de JavaScript se comprueba para determinar si es una instancia de una clase en tiempo de ejecución o una interfaz. Si lo es, y el valor original de Windows en tiempo de ejecución implementa la interfaz, el objeto se pasa a Windows en tiempo de ejecución.|  
|Clases en tiempo de ejecución|`Object`|Los objetos de Windows Runtime pueden ser instancias de clases Runtime que implementan una o más interfaces. Los objetos de Windows en tiempo de ejecución se representan en JavaScript como objetos. La unión de métodos, propiedades y eventos definidos en todas las interfaces implementadas desde la clase representa las propiedades con nombre en el prototipo del objeto de JavaScript. Los consumidores de JavaScript pueden tener acceso a cualquier miembro de la clase directamente. Los objetos de Windows Runtime tienen un prototipo que contiene todos los miembros de todas las interfaces implementadas de la clase Runtime.|
